# About the users

## Who uses the application?

The application has two user roles:

- **New Employee**
  - Completes assigned onboarding tasks
  - Views onboarding information and training materials
  - Can access general company information (colleagues' email addresses)
  - Can use the application before the first working day

- **HR**
  - Creates and maintains onboarding tasks
  - Assigns tasks to employees
  - Manages employee accounts (or relies on automatic account creation through integrations with internal employee management systems)
  - Updates onboarding content when issues are reported

The implemented solution includes only the **New Employee** interface. HR features are part of the final product.


# About the data

## When is the information added?

- When company tools or required training change
- Tasks are generic and reused across employees

## What happens if information is missing or incorrect?

- HR reviews and updates the affected onboarding content

# About the context

## What device does the new employee use?

- A personal laptop before receiving company equipment

## Do they have access before the first working day?

- Yes. HR provides login credentials before the start date so that employees can access general company information
