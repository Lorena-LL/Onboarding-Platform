When writing the frontend, I implemented at the same time some components with the pages using them so that I could test it in the UI and get a good picture of how it will look once integrated. I realized that for some components I should have focused on the implementation of the component and placed it in a more general context, like an empty page that can be ignored when uploading my code to github. By not separating the implementation, I found myself losing time trying to choose which files belong in the commit and what files only affect the page that still needs work done. I decided to be more mindful of that before choosing to implement multiple components at once and to do so if they are highly dependent and if that is the best choice.


Another thing I would like to mention is that it was challenging to manage the time for all the work I wanted to do initially. In the beginning, I had planned that the calendar feature would be done for the demo, at least as a proof of concept, but I realized midway that I didn't have enough time for that. It was really important for me to take a moment and reassess the importance of each feature and decide what I would sacrifice. And I decided that I needed to take the time to write clean code and deliver working functionalities. I think that it's really important to remember as a developer to look at the big picture and reflect on the work you are doing even when constrained by time.
